# Карта проекта Jules (Restaurant Management System)

Данный документ описывает структуру приложения, маршруты фронтенда и эндпоинты бэкенда.

## 1. Frontend (React + Vite + MUI)

### Основные маршруты (Routes)
- `/login` — Страница авторизации сотрудников.
- `/client` — Клиентский портал (Бронирование столов и предзаказ блюд).
- `/waiter` — Интерфейс официанта (Сетка столов, управление статусами, принятие заказов).
- `/kitchen` — Интерфейс кухни (Список активных заказов, управление готовностью).
- `/admin` — Управление залом (Карта столов, список текущих броней).
- `/management` — Управление меню (Категории, блюда, цены, изображения).
- `/users` — Управление персоналом (Список сотрудников, роли, график смен).
- `/stats` — Статистика и аналитика (Выручка, популярные блюда, загрузка зала).

### Ключевые компоненты (src/components)
- `Table.tsx` — Визуализация столика со стульями (меняет цвет и форму в зависимости от статуса).
- `BookingForm.tsx` — Форма создания бронирования (с выбором клиента по телефону).
- `BookingList.tsx` — Список текущих бронирований.
- `WaiterMenu.tsx` — Меню для выбора блюд официантом.
- `Layout.tsx` — Общая обертка с боковой навигацией.
- `ProtectedRoute.tsx` — Защита маршрутов по ролям пользователей.

## 2. Backend (Hono + PostgreSQL)

### Сущности базы данных (Database Schema)
- `users` — Сотрудники (логин, пароль, роль).
- `roles` — Роли (admin, waiter, cook, manager).
- `clients` — База гостей (телефон, имя).
- `tables` — Столы (номер, количество мест, координаты).
- `bookings` — Бронирования (связь: table + client + time_slots).
- `time_slots` — Временные интервалы (9:00 - 21:00).
- `orders` — Заказы (связь: table + client + status).
- `order_items` — Состав заказа (menu_item + количество).
- `menu_items` — Блюда (название, цена, категория, фото).
- `shifts` — Смены сотрудников.

### API Эндпоинты (REST)
#### Авторизация
- `POST /auth/login` — Вход в систему.
- `GET /auth/me` — Проверка текущей сессии.

#### Столы и Бронирование
- `GET /tables` — Получить все столы.
- `PUT /tables/:id/status` — Изменить статус стола.
- `GET /bookings` — Список активных броней.
- `POST /bookings` — Создать бронь (автоматически создает/ищет клиента).
- `GET /time-slots` — Список доступных интервалов времени.

#### Заказы и Меню
- `GET /menu` — Весь список блюд.
- `POST /menu` — Добавить новое блюдо.
- `GET /orders` — Все активные заказы (для кухни и официантов).
- `POST /orders` — Создать заказ.
- `PUT /orders/:id/status` — Изменить статус (open, cooking, ready, served, closed).

#### Клиенты
- `GET /clients/history?phone=...` — История броней и заказов по номеру телефона.

#### Администрирование
- `GET /users` — Список персонала.
- `GET /shifts` — График смен.
- `GET /stats/daily` — Отчет за день.

## 3. Технологический стек
- **Frontend:** React, TypeScript, Material UI, Lucide React (иконки).
- **Backend:** Node.js, Hono.js (фреймворк), PostgreSQL (БД).
- **Storage:** Локальное хранилище для изображений блюд (`backend/storage`).
- **Auth:** JWT / Bcryptjs.
